# Ecommerce API

Цей проект є бекенд-сервісом для електронної комерції, розробленим на Go з використанням Clean Architecture. Він надає RESTful API для управління товарами, кошиком покупок, замовленнями, а також функціонал автентифікації та авторизації.

## Особливості

- **Автентифікація та Авторизація:**
  - Реєстрація користувачів з валідацією та хешуванням паролів.
  - Вхід в систему з генерацією JWT токенів (Access та Refresh).
  - Механізм оновлення токенів.
  - Розмежування ролей (customer, admin).
  - Захищені маршрути з перевіркою токенів та ролей.
- **Управління товарами:**
  - Перегляд каталогу товарів з пагінацією, пошуком та фільтрацією (за категорією, ціною, наявністю).
  - Детальна інформація про товар.
  - CRUD операції для товарів (доступно лише адміністраторам).
  - Контроль залишків на складі.
- **Кошик покупок:**
  - Додавання товарів до кошика з перевіркою наявності.
  - Оновлення кількості та видалення товарів з кошика.
  - Перегляд вмісту кошика з підрахунком загальної вартості.
  - Збереження кошика між сесіями для авторизованих користувачів.
- **Оформлення замовлень:**
  - Створення замовлення з кошика з вказанням адреси доставки та способу оплати.
  - Автоматичне зменшення залишків товарів.
  - Історія замовлень користувача.
  - Статуси замовлень (pending, confirmed, shipped, delivered, cancelled).
  - Скасування замовлення з поверненням товарів на склад.
- **Адміністративна панель:**
  - Управління товарами (CRUD).
  - Перегляд всіх замовлень та зміна їхніх статусів.
  - Перегляд списку користувачів.
  - Статистика (загальний дохід, кількість замовлень, топ товари).

## Технології

- **Backend:** Go (Golang)
- **Веб-фреймворк:** Chi
- **База даних:** PostgreSQL
- **ORM/DB Driver:** sqlx, lib/pq
- **Автентифікація:** golang-jwt/jwt
- **Хешування паролів:** bcrypt
- **Управління залежностями:** Go Modules
- **Контейнеризація:** Docker, Docker Compose
- **API Документація:** Swagger (swaggo)
- **Змінні середовища:** godotenv

## Налаштування та запуск

### Клонування репозиторію

```
git clone https://github.com/MaksimBBB/ecommerce.git
cd ecommerce-api
```

### Файл `.env`

Створіть файл `.env` у корені проекту, скопіювавши вміст з `.env.example` та заповнивши його своїми значеннями.

### Запуск за допомогою Docker Compose

Проект можна запустити у двох режимах: `development` (з hot-reloading) та `production`.

#### Режим розробки (Development)

```
docker compose --profile dev up --build
```

Ця команда збереже образи, запустить контейнери для бази даних та API, а також автоматично перезавантажуватиме сервер при змінах у коді.

#### Продакшн режим (Production)

```
docker compose --profile prod up --build -d
```

Ця команда збереже оптимізовані образи, запустить контейнери у фоновому режимі.

### Заповнення бази даних (Seeding)

Після першого запуску бази даних, ви можете заповнити її початковими даними, виконавши SQL-скрипт:

```
# Запустіть контейнер бази даних, якщо він ще не запущений
docker compose up -d db

# Виконайте seed-скрипт
docker exec -i ecommerce_db psql -U postgres -d ecommerce_db < script/seed.sql
```

**Примітка:** Пароль для всіх тестових користувачів у `seed.sql` - `Test1234!`.

## Структура проекту

Проект дотримується принципів Clean Architecture:

```
ecommerce-api/
├── cmd/              # Точка входу в додаток (main.go)
│   └── api/
├── internal/         # Внутрішній код додатка
│   ├── db/           # Налаштування з'єднання з базою даних
│   ├── domain/       # Доменні сутності (моделі)
│   ├── handler/      # HTTP-хендлери та middleware
│   │   └── http/
│   ├── repository/   # Інтерфейси репозиторіїв та їх реалізації (PostgreSQL)
│   │   └── postgres/
│   └── service/      # Бізнес-логіка (сервіси)
│       ├── auth/
│       ├── cart/
│       ├── order/
│       ├── product/
│       └── user/
├── .migrations/      # SQL-міграції бази даних
├── script/           # Допоміжні скрипти (наприклад, seed.sql)
├── docs/             # Згенерована Swagger документація
├── Dockerfile        # Dockerfile для production
├── Dockerfile.dev    # Dockerfile для development
├── docker-compose.yml# Конфігурація Docker Compose
├── go.mod            # Go модулі
├── go.sum            # Go суми
└── README.md         # Цей файл
